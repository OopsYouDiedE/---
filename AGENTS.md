# 《新六艺》项目协作规范

## 1. 内容工作流

涉及事实、科学方法、健康建议、训练方法、技能教学时，必须按以下顺序工作：

**研究 → 写入知识库 → 写正文/修正文**

- 不得跳过研究直接写指导性内容。
- 正文覆盖的每个具体领域，知识库必须有对应文件；相邻领域不能替代直接领域。
- 任何方法、步骤、标准都要在书中给出可执行内容，不能只给链接或让读者自行查询。
- 方法为，开启SubAgent，然后等待返回完整的内容才能编写。

## 2. 证据要求

知识检索内容必须交叉核实 2-3 个权威来源。优先级：

1. 同行评审研究和系统综述
2. 专业机构官方指南
3. 专业机构教育材料
4. 军事、警察、救援机构公开训练材料

新增事实进入正文前，先进入 `knowledge/`。

## 3. 受众完整性

指导性内容必须说明适用人群，并覆盖必要修正，如果该章节面向目标为全年龄段，那么就必须考虑到不同人群的策略。这里以第一章为例：

- 体能基础：零基础、有运动习惯、有竞技经验
- 身体状况：正常体重、超重/肥胖、慢性病或关节问题
- 性别和年龄：女性和青少年
- 资源条件：无器械/有器械，无场地/有场地

如果某方案只适用于健康成年人，必须明说，并补充其他群体的起步或禁忌方案。

## 4. 本书定位

本书是被认为内容可靠的宣传倡议书，不是严肃期刊，也不是百科全书。

写作目标是让读者：

- 相信这件事与自己有关；
- 知道为什么要做；
- 读完后能立刻开始做一点具体的事。

章节通常遵循：真实案例 → 打破直觉 → 权威框架 → 细颗粒操作指南 → 社会维度升华。高信息密度可以保留，但不得牺牲行动入口和安全边界。

## 5. 配图标注

Markdown 正文中用以下格式标记配图需求：

```md
<!-- [配图] 具体描述：图片内容、角度、重点说明的信息 -->
```

描述要具体到设计师或摄影师能直接执行。进入渲染阶段后，图片是否替换、缩放、改成图表，由渲染脚本决定。

## 6. 渲染阶段规则

章节进入 PDF/LaTeX 渲染阶段后：

- Markdown 是内容母本。
- 渲染脚本是版式主源。
- 标题层级、提示框类别、图片替换、表格改写、分页控制、图注和引用排版，以渲染脚本为准。
- 渲染脚本可压缩表达、合并重复、调整展示顺序，但不得改变事实含义，不得新增未入知识库的专业事实，不得删除适用条件和安全边界。

当前渲染流程：

1. 编辑 Markdown 内容和 `knowledge/` 资料。
2. 在 `tmp/pdfs/` 编写或修改渲染脚本。
3. 图片放入 `assets/chapter-XX/...`，正文图位由脚本映射到具体图片。
4. 用 XeLaTeX 渲染中文 PDF；常用输出目录为 `output/pdf/...`。
5. 用 Poppler `pdftoppm` 把 PDF 渲染成 PNG 页面。
6. 逐页检查 PNG：标题孤立、段落拆页、提示框断裂、表格溢出、图片含义不清、页尾大面积空白都必须修。

版式约定：

- 提示框标题统一为三类：`注意`、`要点`、`行动`。
- 警告、年龄修正、医疗禁忌、超重修正使用淡红色。
- 重点原则使用蓝色；行动入口、班组任务使用绿色。
- 宁可把整段或整个模块挪到下一页，也不要在段落内部硬拆。
- 这是一本书，页面要干净、有主次，不按期刊论文排。
