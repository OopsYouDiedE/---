## 1. 项目定位

本节规定三件事：内容深度、针对群体、写作目的。

- 内容深度：以 16-24 岁读者为准。面向 10-18 岁读者的起步路径与禁忌，以分层补充承载；二者都属正式内容（分层办法见 §6）。
- 针对群体：主要面向 16-24 岁、其次面向 10-18 岁的青少年。读者的具体画像见 §3。
- 写作目的：培养具有协同能力、互助能力与独立思考能力，能够理解现代社会基本运行的公民。以君子六艺为文化框架组织内容，以实践为驱动。

优先级声明：本节是身份定位。当条款之间发生取舍时，可执行判据以 §2、§6、§11 为准；当内容深度与行动入口或安全边界发生取舍时，行动入口与安全边界优先。涉及「团结、组织性」的表述，合规边界以 §11 为准。

## 2. 本书定位

本书是内容可靠的公民自我培养倡议书，写作标准高于日常传播材料、低于严肃期刊，体例上与百科全书不同。

本书的主张通过选材与内容本身传递：读者读完之后，凭书中呈现的事实、案例与推理，自行认识到作者想说的事。正文在开头呈现具体处境与证据，不堆砌政治表态或口号，也不预先宣布结论。作者的角色是呈现证据和推理，结论由读者得出（语气规范见 §7）。

写作目标是让读者：

- 相信这件事与自己有关；
- 知道为什么要做；
- 读完后能开始做一点具体的事，或做出一个具体判断。

章节通常遵循：真实案例 → 打破直觉 → 权威框架 → 细颗粒操作指南 → 回到读者自身的处境与可做的下一步。规程类、定义类、元反思类章节（如礼节、事实核验、终章）可省略「真实案例 → 打破直觉」开场，直接给规程、定义或判断，不为凑戏剧化开场而编造（呼应 §4）；这类章节的转化目标可以是「做出一个具体判断」而非「做一个具体动作」。

高信息密度可以保留，前提是守住行动入口和安全边界。

## 3. 目标读者画像

读者画像依据全书现有成稿内容归纳，随内容更新。

现有四艺覆盖：身体胜任与危机行动力（射）、移动与协作中的德行与组织（御）、公共交往与组织运行（礼）、准确表达与事实核验（书）。据此，本书服务的青少年读者具有以下需求：

- 希望获得课堂之外可实操的身体与危机应对能力；
- 希望学会与他人协作、组织一件事、在群体中承担角色；
- 关心公共事务与集体行动中的公平与秩序；
- 希望具备核查事实、识别失真信息的判断力；
- 有行动意愿，但缺少可依循的框架和同类群体。

写作对象是上述青少年读者。付费者与传播支持者是另一类受众，其关切用于选题与传播，不替代写作对象。书中内容针对青少年读者的上述需求设计。

## 4. 内容工作流

所有涉及专业领域的内容（体能训练、急救、灾害应急、军事素养、载具与交通、组织与物流、契约与礼节、事实核验与论文写作，以及后续的乐理、数理等），必须基于官方教程、权威机构发布的实践指南或有据可查的实践记录。禁止凭印象编造细节。

涉及事实、科学方法、健康建议、训练方法、技能教学时，按以下顺序工作：

**研究 → 写入知识库 → 写正文/修正文**

- 不得跳过研究直接写指导性内容。
- 正文覆盖的每个具体领域，知识库必须有对应文件；相邻领域不能替代直接领域。
- 任何方法、步骤、标准都要在书中给出可执行内容，不能只给链接或让读者自行查询。
- 方法为，开启 SubAgent，等待返回完整的内容后再编写。
- 事后对照审计：指导性正文定稿后，须逐段对照知识库标注四种状态（来源可靠／存疑／无源／与源不符）。无源与不符项不得留在指导性正文。

## 5. 证据要求与知识管理

知识检索内容必须交叉核实 2-3 个权威来源。优先级：

1. 同行评审研究和系统综述
2. 专业机构官方指南
3. 专业机构教育材料
4. 军事、警察、救援机构公开训练材料

来源与引用规范：

- 「至少两个来源」指至少两个相互独立的来源。同一上游被二次转述、换算或再引用，只计为一个来源。
- 中国单一官方口径的权威内容（如 GA 标准、防空警报信号、黄金 72 小时自救互救概念等），可直接引用，注明「单一权威源」即可，不因凑不满两源被挡在正文外。
- 「待证实」分两级：一级是「可引用但注明局限」（单一来源，或口径未统一），可进正文并注明局限；二级是「不得引用」（未经核实），不得进入正式写作。

知识管理规范：

- 通过网络检索采集到的事实性内容，统一记录至项目根目录下的 `knowledge/` 文件夹，每条以其主题作为文件名，使用 Markdown 格式存储。
- 记录原始信源的名称与链接。
- 每条事实至少两个相互独立的可信信源交叉证实，并在条目中标注各信源。
- 新增事实进入正文前，先进入 `knowledge/`。

## 6. 受众完整性

指导性内容必须说明适用人群，并覆盖必要修正。

每章按其领域自定受众维度。通用维度骨架：

- 经验层级：零基础、有经验、熟练；
- 协作规模：个人、小组、组织；
- 身心与年龄：性别、青少年、慢性病或身体限制；
- 场景与资源：有无器械、有无场地、时间与环境约束。

> 示例：射卷体能章的受众维度落为体能基础、身体状况、性别年龄、资源条件；礼、书、御章的受众维度多落在个人／小组／组织、无经验／有协作经验。各章按本骨架取用适配的维度，不必逐维展开。

按需触发：仅当某方法对某人群从「打折扣」变为「不适用或有害」时，才必须给出该人群的适用标注与修正；不产生此类差异的维度无需机械展开。

若某方案只适用于健康成年人或特定条件人群，必须明说；对不存在安全起步路径的人群，标注不建议并给出替代目标，不硬凑降级起步方案。青少年是主受众，正文默认按主受众写作。

## 7. 文风与语气规范

### 句式

- 多用肯定句式，少用否定句式。
- 少用复杂从句，保持句子结构简洁直接。
- 对比否定结构（「不是……而是……」「并非……而是……」）用于澄清关键误解，一节正文不超过 2 处，不作为默认句式连续堆砌。
- 以书面语为底，允许第二人称直接称呼和短句口语节奏来降低门槛；避免网络流行语、语气词堆砌和随意省略。

### 修辞

- 优先使用事实陈述和逻辑推演，少用比喻。
- 不使用专业公式或学术化的表述方式。
- 行文应当制造「我看我也行」的参与感和可达性，降低读者的心理门槛。

### 态度

- 只摆事实，只做逻辑论证。不说教，不劝诫，不布道。
- 结论由读者自行得出。作者的角色是呈现证据和推理，而非宣布答案。
- 整体语气克制、冷静、精确。

### 落地检查

- 渲染阶段增加语气检查项：统计每章对比否定结构出现次数、口语化词命中数，超出本节尺度即标记复核（检查脚本另行实现，纳入 §9 逐页检查清单）。

## 8. 配图标注

Markdown 正文中用以下格式标记配图需求：

```md
<!-- [配图] 具体描述：图片内容、角度、重点说明的信息 -->
```

该标记是配图的意图声明入口，描述要具体到设计师或摄影师能直接执行。配图生成、插图审核清单、命名入库等操作细则，见《GPT渲染与插图工作流规范.md》（§8/§9 的下游补丁）。

配图标记只声明意图，不构成「安全禁忌随注释自动随图走」的保证。图内数字、禁忌与适用条件的最终守门，在渲染规范的出图审核清单与图内标签要求。

进入渲染阶段后，图片是否替换、缩放、改成图表，由渲染脚本决定。渲染时须满足硬验收：命中的配图位数等于渲染数据表中非空条目数，任一图位失配即报错，不得静默丢图（见 §9）。

## 9. 渲染阶段规则

章节进入 PDF/LaTeX 渲染阶段后：

- Markdown 是内容母本。
- 渲染脚本是版式主源。
- 标题层级、提示框类别、图片替换、表格改写、分页控制、图注和引用排版，以渲染脚本为准。
- 渲染脚本可压缩表达、合并重复、调整展示顺序，但不得改变事实含义，不得新增未入知识库的专业事实，不得删除适用条件和安全边界。

编址与命名（全书无全局章号，一律用「艺_序」复合编号）：

- 图片放入 `assets/{艺}_{序}/...`（如 `assets/she-01/`），正文图位由脚本映射到具体图片。
- 输出目录 `output/pdf/{艺}_{序}/...`。
- 渲染脚本按艺_序命名（如 `render_she-01.py`）。

当前渲染流程（截至 ch01/ch02 已验证，其余章比照执行）：

1. 编辑 Markdown 内容和 `knowledge/` 资料。
2. 在 `tmp/pdfs/` 编写或修改渲染脚本。
3. 图片放入 `assets/{艺}_{序}/...`，正文图位由脚本映射到具体图片。
4. 用 XeLaTeX 渲染中文 PDF；常用输出目录为 `output/pdf/...`。
5. 用 Poppler `pdftoppm` 把 PDF 渲染成 PNG 页面。
6. 逐页检查 PNG：标题孤立、段落拆页、提示框断裂、表格溢出、图片含义不清、页尾大面积空白都必须修；并执行 §7 的语气检查项。

构建期断言（把安全边界从人工承诺变成机器校验）：

- 脚本启动先断言源文件（SRC）存在。
- 提示框与配图的关键词映射表，每个键必须在正文命中至少一次；任一键失配即报错退出，不得让安全提示静默降级为普通段落。
- 正文改标题、改文件名、增删医疗／法律／年龄／安全类提示时，必须同步更新渲染数据并跑通断言。

版式约定：

- 提示框类别在正文层用统一标签标注（见 §13），渲染脚本只做样式映射，不做语义分类。三类标签统一为：`注意`、`要点`、`行动`。
- 提示框颜色按领域无关的语义分级：安全、禁忌、医疗、年龄与身体限制类用淡红色；核心原则用蓝色；行动入口、协作任务用绿色。红色提示区分「致命」与「一般」两级，避免满页红框淹没最需要看见的一两条。
- 各艺按上述语义自配本领域的提示框名称。
- 宁可把整段或整个模块挪到下一页，也不要在段落内部硬拆。
- 这是一本书，页面要干净、有主次，不按期刊论文排。

## 10. 并行任务与模型分配

凡是互不依赖、不需共享中间结果的子任务（多章节研究、多知识库条目采集、多页面配图检查、多脚本渲染等），可拆分为并行 SubAgent 执行以提升吞吐。铺量不优先于把关；单人把关带宽是硬约束，不得为并行而并行制造超出可审计量的完整稿。

- 可并行的判断标准：子任务之间没有先后依赖、不需要共享中间结果，才可并行；有依赖关系的步骤（如先研究后写正文）按顺序执行。
- 往返协作类别：正反方互读、共享彼此完整产出、按串行往返推进的对抗-裁决式协作（如设计纲领的正反辩论），允许并鼓励共享中间结果，不受「可并行」准入标准约束。
- 模型选择按内容判断密度就高不就低，不得所有子任务统一用同一档模型：
  - 简单检索、格式整理、单条事实核对、机械性重复劳动 → 轻量模型（Haiku）。
  - 常规知识库写入、单节正文撰写、单页排版检查 → 中档模型（Sonnet）。
  - 跨章节一致性判断、复杂论证结构设计、政治边界把关、渲染版式整体决策、嵌在正文里的隐藏前提判断 → 重型模型（Opus）。
- 交付与质量分开：SubAgent 返回完整结果只是交付条件，不等于内容正确；承重事实仍须经 §5 交叉核实与 §4 事后对照审计。（「等完整返回再编写」的纪律见 §4，本节不重复。）
- 跨艺一致性关卡：每写完一艺，发一个 Opus SubAgent 对照 §13 的一致性契约做横向比对，产出可勾除清单。

## 11. 政治边界

- 本书的定位是现代公民的自我培养指南。所有内容服务于这一定位。
- 避免任何可被解读为推翻政权或策划组织性运动的措辞。
- 引用外国的军事动员、对抗或组织性动员案例前，须通过「落点测试」：案例的叙事落点是平民疏散、自救互救或民防，则可用；落点是国家间军事对峙或政权动员，则不用。此测试适用于所有外国案例，包括不在下条名单中的国家（如以色列、瑞士等）。
- 硬底线：在军事与政治领域，不使用日本和台湾的案例；自然灾害与应急救援领域除外。落点测试与本条同时生效，二者取严。
- 本条管辖范围覆盖知识库采集与写作建议：`knowledge/` 条目涉及受限案例时，就地标注「仅供背景参照，不得进入正文」，不得写「建议正文采用」一类使用提示。
- 引用中国政府官方数据和官方承认的逻辑（如「黄金 72 小时」自救互救概念）作为论证基础时，以客观引述的方式呈现。

## 12. 章节生命周期与状态管理

章节状态分四态，外加一条重构废弃旁支：

- 草案：只有大纲或要点，正文未写（如乐、数当前状态）。
- 成稿：正文写完。毕业判据——知识库对应文件齐备、按 §7 与 §11 校对通过、已回写全书大纲一览。
- 待渲染：成稿已登记，等待渲染。
- 已渲染：通过 §9 渲染流程与逐页检查。
- 重构废弃：被重构方案取代的旧稿（如射卷 5→4 中退役的旧章），执行完毕即归档。

单一真相源：`全书大纲一览.md` 是全书章节结构与状态的权威来源。任何章节的正文增删改，须在同一提交内回写该文件。

过程文件：写作大纲、重设计方案、大纲细化等过程文件，写完即移入 `drafts/` 或 `archive/` 子目录；根目录只留成稿正文与治理文件。

未开工的艺（乐、数）在真相源中显式标注「未开工」或「在途」，使欠债在台账上可见。

## 13. 跨章一致性契约

四艺由不同 SubAgent 在不同时间生产，术语、案例、版式、提示框标签会漂移。本节约定三张表，作为跨章一致性的单一依据。

### 提示框标签白名单

### 统一术语与人称表
