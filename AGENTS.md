# 《新六艺》项目协作规范

## 1. 项目定位

本书是一本主要面向 16-24 岁、其次面向 10-18 岁青少年的现代公民自我培养指南，以君子六艺为文化引子，以实践为驱动，强调组织能力、协同能力与独立思考。内容深度以 16-24 岁读者为天花板设定，不为迁就低龄读者而降低深度；面向低龄读者的调整以补充说明的方式处理（见第 6 节）。

核心要素：培养一个具有组织和团结能力，能够对现代社会具备基本理解的公民。

## 2. 本书定位

本书是被认为内容可靠的宣传倡议书，不是严肃期刊，也不是百科全书。

写作目标是让读者：

- 相信这件事与自己有关；
- 知道为什么要做；
- 读完后能立刻开始做一点具体的事。

章节通常遵循：真实案例 → 打破直觉 → 权威框架 → 细颗粒操作指南 → 回到读者自身的处境与可做的下一步。全书语气与章节收尾完全以第 7 节《文风与语气规范》为准：不拔高、不升华、不布道，结论由读者自行得出。高信息密度可以保留，但不得牺牲行动入口和安全边界。

## 3. 目标读者画像

本书的付费读者和潜在支持者具有以下特征：

- 对原子化社会中人与人之间的疏离和脆弱感到担忧
- 关注军事与政治议题
- 对当前中国教育体系存在明确不满
- 有行动意愿，但缺少组织框架和同类群体

书中内容应针对这一群体的核心痛点进行设计。

## 4. 内容工作流

所有涉及专业领域的内容（体能训练、急救、灾害应急、军事素养等），必须基于官方教程、权威机构发布的实践指南或有据可查的实践记录。禁止凭印象编造细节。

涉及事实、科学方法、健康建议、训练方法、技能教学时，必须按以下顺序工作：

**研究 → 写入知识库 → 写正文/修正文**

- 不得跳过研究直接写指导性内容。
- 正文覆盖的每个具体领域，知识库必须有对应文件；相邻领域不能替代直接领域。
- 任何方法、步骤、标准都要在书中给出可执行内容，不能只给链接或让读者自行查询。
- 方法为，开启 SubAgent，然后等待返回完整的内容才能编写。

## 5. 证据要求与知识管理

知识检索内容必须交叉核实 2-3 个权威来源。优先级：

1. 同行评审研究和系统综述
2. 专业机构官方指南
3. 专业机构教育材料
4. 军事、警察、救援机构公开训练材料

知识管理规范：

- 通过网络检索采集到的事实性内容，统一记录至项目根目录下的 `knowledge/` 文件夹，每条以其主题作为文件名，使用 Markdown 格式存储。
- 记录原始信源的名称与链接。
- 每条事实至少两个可信信源交叉证实，并在条目中标注各信源。
- 未经交叉证实的内容标注「待证实」，正式写作中不得直接引用。
- 新增事实进入正文前，先进入 `knowledge/`。

## 6. 受众完整性

指导性内容必须说明适用人群，并覆盖必要修正，如果该章节面向目标为全年龄段，那么就必须考虑到不同人群的策略。这里以第一章为例：

- 体能基础：零基础、有运动习惯、有竞技经验
- 身体状况：正常体重、超重/肥胖、慢性病或关节问题
- 性别和年龄：女性和青少年
- 资源条件：无器械/有器械，无场地/有场地

如果某方案只适用于健康成年人，必须明说，并补充其他群体的起步或禁忌方案。

## 7. 文风与语气规范

### 句式

- 多用肯定句式，少用否定句式。
- 少用复杂从句，保持句子结构简洁直接。
- 避免频繁使用「不是……而是……」「并非……而是……」等对比否定结构。
- 使用书面语，拒绝口语化表达。

### 修辞

- 少用比喻，优先使用事实陈述和逻辑推演。
- 不使用专业公式或学术化的表述方式。
- 行文应当制造「我看我也行」的参与感和可达性，降低读者的心理门槛。

### 态度

- 只摆事实，只做逻辑论证。不说教，不劝诫，不布道。
- 结论由读者自行得出。作者的角色是呈现证据和推理，而非宣布答案。
- 整体语气应当克制、冷静、精确，如同侦探在陈述调查结果。

## 8. 配图标注

Markdown 正文中用以下格式标记配图需求：

```md
<!-- [配图] 具体描述：图片内容、角度、重点说明的信息 -->
```

描述要具体到设计师或摄影师能直接执行。进入渲染阶段后，图片是否替换、缩放、改成图表，由渲染脚本决定。

## 9. 渲染阶段规则

章节进入 PDF/LaTeX 渲染阶段后：

- Markdown 是内容母本。
- 渲染脚本是版式主源。
- 标题层级、提示框类别、图片替换、表格改写、分页控制、图注和引用排版，以渲染脚本为准。
- 渲染脚本可压缩表达、合并重复、调整展示顺序，但不得改变事实含义，不得新增未入知识库的专业事实，不得删除适用条件和安全边界。

当前渲染流程：

1. 编辑 Markdown 内容和 `knowledge/` 资料。
2. 在 `tmp/pdfs/` 编写或修改渲染脚本。
3. 图片放入 `assets/chapter-XX/...`，正文图位由脚本映射到具体图片。
4. 用 XeLaTeX 渲染中文 PDF；常用输出目录为 `output/pdf/...`。
5. 用 Poppler `pdftoppm` 把 PDF 渲染成 PNG 页面。
6. 逐页检查 PNG：标题孤立、段落拆页、提示框断裂、表格溢出、图片含义不清、页尾大面积空白都必须修。

版式约定：

- 提示框标题统一为三类：`注意`、`要点`、`行动`。
- 警告、年龄修正、医疗禁忌、超重修正使用淡红色。
- 重点原则使用蓝色；行动入口、班组任务使用绿色。
- 宁可把整段或整个模块挪到下一页，也不要在段落内部硬拆。
- 这是一本书，页面要干净、有主次，不按期刊论文排。

## 10. 并行任务与模型分配

凡是可以拆分为多个互不依赖子任务的工作（多章节研究、多知识库条目采集、多页面配图检查、多脚本渲染等），必须全部拆开为并行 SubAgent 执行，禁止在主线程里逐个串行处理。

- 判断标准：子任务之间没有先后依赖、不需要共享中间结果，才可并行；有依赖关系的步骤（如先研究后写正文）仍按顺序执行，不得强行并行。
- 模型选择必须匹配任务难度，不得所有子任务统一用同一档模型：
  - 简单检索、格式整理、单条事实核对、机械性重复劳动 → 轻量模型（Haiku）。
  - 常规知识库写入、单节正文撰写、单页排版检查 → 中档模型（Sonnet）。
  - 跨章节一致性判断、复杂论证结构设计、政治边界把关、渲染版式整体决策 → 重型模型（Opus）。
- 并行发出的每个 SubAgent 必须等待其返回完整结果后，才能据此继续写作、渲染或整合，不得用未返回的中间状态拼凑正文。

## 11. 政治边界

- 本书的定位是现代公民的自我培养指南。所有内容服务于这一定位。
- 避免任何可被解读为推翻政权或策划组织性运动的措辞。
- 引用外国案例时，除自然灾害与应急救援领域外，不得使用日本和台湾的案例。
- 引用中国政府官方数据和官方承认的逻辑（如「黄金72小时」自救互救概念）作为论证基础时，以客观引述的方式呈现。
