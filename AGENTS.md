## 1. 项目定位

这本书写给 16–24 岁（主）和 10–18 岁（次）的青少年，目的是培养有协同能力、互助能力、独立思考能力，能理解现代社会基本运行的公民。文化框架用君子六艺，内容以实践驱动。

- 正文深度按 16–24 岁读者写。10–18 岁读者需要的起步路径与禁忌，用分层补充承载；分层内容与正文同为正式内容（分层办法见 §6）。
- 读者的具体画像见 §3。

本节只回答「这本书是什么」。具体写作中条款打架时，按 §2（写法）、§6（受众）、§11（政治边界）的可执行判据裁决；内容深度与行动入口或安全边界冲突时，行动入口与安全边界优先；凡涉及「团结、组织性」的表述，一律以 §11 划界。

## 2. 本书定位

这是一本内容可靠的公民自我培养倡议书。写作标准高于日常传播材料、低于严肃期刊；体例是书，不是百科全书。

本书不喊口号。主张通过选材与内容本身传递：正文先摆具体处境与证据，不预先宣布结论，也不堆政治表态；读者读完，凭书里的事实、案例与推理，自己认识到作者想说的事。作者只负责呈现证据和推理（语气规范见 §7）。

写作要达到三个效果——读者：

- 相信这件事与自己有关；
- 知道为什么要做；
- 读完能开始做一点具体的事，或做出一个具体判断。

章节的常规结构是：真实案例 → 打破直觉 → 权威框架 → 细颗粒操作指南 → 回到读者自身的处境与可做的下一步。规程类、定义类、元反思类章节（如礼节、事实核验、终章）可以省去「真实案例 → 打破直觉」的开场，直接给规程、定义或判断——不为凑戏剧化开场而编造（呼应 §4）；这类章节的转化目标可以是「做出一个判断」，不必是「做一个动作」。

信息密度可以高，前提是行动入口和安全边界不被挤掉。

## 3. 目标读者画像

读者画像从全书现有成稿内容归纳，随内容更新。

现有四艺 17 章成稿，分别覆盖：

- 射——身体胜任与危机行动力：体能建设、人身防卫、军事基础、应急预案；
- 御——移动的德行与协作的组织：载具安全、防御性驾驶、物流调度、安全搬运；
- 礼——人际对等与组织运行：体贴他人、为少数群体提供便利、公平契约、礼节、组织怎么运行；
- 书——准确表达与事实核验：描述事实、书写论文、核验信息、识别汇报中的谎言。

对应地，本书服务的青少年读者是这样一群人：

- 想获得课堂之外可实操的身体与危机应对能力；
- 想学会与人协作、组织一件事，也想看懂学校、公司、团队这些组织和身边系统的运转规律；
- 关心借钱、分摊、约定、排队这些日常利益往来中的公平，想不吃亏也不伤关系；
- 想在陌生或正式的社交场合读懂隐性规则、举止得体，而不靠死记条目；
- 想学会准确描述事实、写出经得起质疑的论证，也想具备核查信息、识别失真与谎言的判断力；
- 想学会在日常与危机中照顾和保护他人——从让身边人省力，到守护比自己弱势的人；
- 有行动意愿，但缺少可依循的框架和同类群体。

写作对象就是这群读者。付费者与传播支持者是另一类受众：他们的关切用于选题与传播决策，但不改变这本书是写给谁的。

## 4. 内容工作流

所有专业领域的内容（体能训练、急救、灾害应急、军事素养、载具与交通、组织与物流、契约与礼节、事实核验与论文写作，以及将来的乐理、数理），必须有据可查：官方教程、权威机构发布的实践指南、或可查证的实践记录。禁止凭印象编细节。

凡涉及事实、科学方法、健康建议、训练方法、技能教学，固定按这个顺序做：

**研究 → 写入知识库 → 写正文/修正文**

- 不许跳过研究直接写指导性内容。
- 正文覆盖的每个具体领域，知识库必须有对应文件；相邻领域的文件不能顶替直接领域的文件。
- 书里给出的每个方法、步骤、标准都要能直接执行，不能只丢链接或让读者自己去查。
- 研究交给 SubAgent 执行；等它返回完整内容之后再动笔，不拿半截结果开写。
- 事后审计：指导性正文定稿后，逐段对照知识库，标注四种状态之一——来源可靠／存疑／无源／与源不符。无源和不符的内容不得留在指导性正文里。

## 5. 证据要求与知识管理

检索到的内容要用 2–3 个权威来源交叉核实。来源按可信度排序：

1. 同行评审研究和系统综述
2. 专业机构官方指南
3. 专业机构教育材料
4. 军事、警察、救援机构公开的训练材料

引用规则：

- 「至少两个来源」指相互独立的来源。同一个上游被二次转述、换算、再引用，只算一个。
- 中国单一官方口径的权威内容（GA 标准、防空警报信号、「黄金 72 小时」自救互救概念等），可以直接引用，标注「单一权威源」即可——不必为凑第二个来源把它挡在正文外。
- 「待证实」分两级：一级「可引用但注明局限」（单一来源，或口径未统一），可以进正文，注明局限即可；二级「不得引用」（未经核实），不得进入正式写作。

知识库规则：

- 网络检索采集到的事实，统一存进项目根目录的 `knowledge/`，一个主题一个 Markdown 文件，主题即文件名。
- 记录原始信源的名称与链接。
- 每条事实至少两个相互独立的可信信源交叉证实，并在条目里标出各信源。
- 新事实先进 `knowledge/`，再进正文。

## 6. 受众完整性

指导性内容必须说清适用于谁，并覆盖必要的修正方案。

每章按自己的领域选受众维度，通用骨架有四组：

- 经验层级：零基础、有经验、熟练；
- 协作规模：个人、小组、组织；
- 身心与年龄：性别、青少年、慢性病或身体限制；
- 场景与资源：有无器械、有无场地、时间与环境约束。

> 例：射卷体能章的维度落在体能基础、身体状况、性别年龄、资源条件；礼、书、御各章多落在个人／小组／组织、无经验／有协作经验。各章从骨架里取用适配的维度即可，不必四组都展开。

触发标准：只有当某方法对某人群从「效果打折」变成「不适用或有害」时，才必须给该人群写适用标注和修正方案；不产生这种差异的维度，不用机械展开。

如果一个方案只适用于健康成年人或特定条件的人群，必须明说。对不存在安全起步路径的人群，直接标注不建议、给出替代目标，不硬凑一个降级起步方案。青少年是主受众，正文默认按主受众写。

## 7. 文风与语气规范

### 句式

- 多用肯定句，少用否定句。
- 少用复杂从句，句子结构简洁直接。
- 对比否定结构（「不是……而是……」「并非……而是……」）只用来澄清关键误解，一节正文不超过 2 处，不当默认句式连着堆。
- 以书面语为底，允许用第二人称直接称呼、用短句口语节奏降低门槛；不用网络流行语，不堆语气词，不随意省略。

### 修辞

- 优先事实陈述和逻辑推演，少用比喻。
- 不用专业公式，不用学术化表述。
- 行文要制造「我看我也行」的参与感，降低读者的心理门槛。

### 态度

- 只摆事实，只做逻辑论证。不说教，不劝诫，不布道。
- 结论由读者自行得出。作者负责呈现证据和推理，不负责宣布答案。
- 整体语气克制、冷静、精确。

### 落地检查

- 渲染阶段加一道语气检查：统计每章对比否定结构的出现次数、口语化词的命中数，超出本节尺度就标记复核（检查脚本另行实现，并入 §9 的逐页检查清单）。

## 8. 配图标注

正文里用这个格式声明配图需求：

```md
<!-- [配图] 具体描述：图片内容、角度、重点说明的信息 -->
```

描述要具体到设计师或摄影师拿到就能执行。配图怎么生成、怎么审核、怎么命名入库，见《GPT渲染与插图工作流规范.md》（§8/§9 的操作细则）。

两点边界：

- 这个标记只声明意图。图里的数字、禁忌、适用条件不会因为写进注释就自动出现在图上——最终守门在渲染规范的出图审核清单和图内标签要求。
- 进入渲染阶段后，图片是否替换、缩放、改成图表，由渲染脚本决定。但渲染必须满足硬验收：实际命中的配图位数等于渲染数据表里非空条目数，任何一个图位失配就报错，不许静默丢图（见 §9）。

## 9. 渲染阶段规则

章节进入 PDF/LaTeX 渲染阶段后，分工如下：

- Markdown 是内容母本。
- 渲染脚本是版式主源：标题层级、提示框类别、图片替换、表格改写、分页控制、图注和引用排版，都以它为准。
- 渲染脚本可以压缩表达、合并重复、调整展示顺序，但不得改变事实含义，不得新增未入知识库的专业事实，不得删掉适用条件和安全边界。

编号与路径（全书没有全局章号，一律用「艺_序」复合编号）：

- 图片放 `assets/{艺}_{序}/...`（如 `assets/she-01/`），正文图位由脚本映射到具体图片。
- PDF 输出到 `output/pdf/{艺}_{序}/...`。
- 渲染脚本按艺_序命名（如 `render_she-01.py`）。

渲染流程（ch01/ch02 已验证，其余章比照执行）：

1. 编辑 Markdown 正文和 `knowledge/` 资料。
2. 在 `tmp/pdfs/` 写或改渲染脚本。
3. 图片放进 `assets/{艺}_{序}/...`。
4. 用 XeLaTeX 渲染中文 PDF，输出到 `output/pdf/...`。
5. 用 Poppler `pdftoppm` 把 PDF 转成逐页 PNG。
6. 逐页检查 PNG：标题孤立、段落拆页、提示框断裂、表格溢出、图片含义不清、页尾大面积空白，见一处修一处；同时跑 §7 的语气检查。

构建期断言——安全边界不靠人工承诺，靠机器校验：

- 脚本启动先断言源文件（SRC）存在。
- 提示框与配图的关键词映射表，每个键必须在正文命中至少一次；任何一个键失配就报错退出，不允许安全提示静默降级成普通段落。
- 改标题、改文件名、增删医疗／法律／年龄／安全类提示时，必须同步更新渲染数据并跑通断言。

版式约定：

- 提示框类别在正文层用统一标签标注（见 §13），渲染脚本只做样式映射，不做语义分类。标签就三类：`注意`、`要点`、`行动`。
- 提示框颜色按语义分级，与领域无关：安全、禁忌、医疗、年龄与身体限制类用淡红；核心原则用蓝；行动入口、协作任务用绿。红色再分「致命」「一般」两级，避免满页红框淹没最该被看见的那一两条。
- 各艺按这套语义自定本领域的提示框名称。
- 宁可把整段、整个模块挪到下一页，也不在段落内部硬拆。
- 这是一本书：页面要干净、有主次，不按期刊论文排。

## 10. 并行任务与模型分配

互不依赖、不需要共享中间结果的子任务（多章研究、多条知识库采集、多页配图检查、多脚本渲染等），拆成并行 SubAgent 执行，提升吞吐。但产量让位于把关：单人审稿带宽是硬约束，不许为了并行而并行，造出审不过来的完整稿。

- 能不能并行，看两条：子任务之间没有先后依赖、不需要共享中间结果。有依赖的步骤（比如先研究后写正文）老实串行。
- 例外是往返协作：正反方互读、共享彼此完整产出、串行往返推进的对抗-裁决式协作（比如设计纲领的正反辩论），允许并鼓励共享中间结果，不受上面的并行准入标准约束。
- 模型按内容的判断密度选，就高不就低，不许所有子任务一律同档：
  - 简单检索、格式整理、单条事实核对、机械性重复劳动 → 轻量模型（Haiku）。
  - 常规知识库写入、单节正文撰写、单页排版检查 → 中档模型（Sonnet）。
  - 跨章一致性判断、复杂论证结构设计、政治边界把关、渲染版式整体决策、辨认正文里嵌着的隐藏前提 → 重型模型（Opus）。
- 交付不等于正确：SubAgent 返回完整结果只是交付条件，承重事实照样要过 §5 的交叉核实和 §4 的事后审计。（「等完整返回再动笔」的纪律在 §4，此处不重复。）
- 跨艺一致性关卡：每写完一艺，发一个 Opus SubAgent 对照 §13 的三张表做横向比对，产出可勾除的清单。

## 11. 政治边界

- 本书的定位是现代公民的自我培养指南，所有内容服务于这个定位。
- 避免任何可被解读为推翻政权或策划组织性运动的措辞。
- 引用外国的军事动员、对抗或组织性动员案例前，先过「落点测试」：案例的叙事落点是平民疏散、自救互救或民防，可用；落点是国家间军事对峙或政权动员，不用。此测试适用于所有外国案例，包括不在下一条名单里的国家（如以色列、瑞士）。
- 硬底线：军事与政治领域不使用日本和台湾的案例；自然灾害与应急救援领域除外。落点测试与本条同时生效，二者取严。
- 本节同样管知识库采集和写作建议：`knowledge/` 条目涉及受限案例时，就地标注「仅供背景参照，不得进入正文」，不得写「建议正文采用」这类使用提示。
- 引用中国政府官方数据和官方承认的逻辑（如「黄金 72 小时」自救互救概念）作为论证基础时，以客观引述的方式呈现。

## 12. 章节生命周期与状态管理

章节状态有四种，另有一个重构专用的旁支状态：

- 草案：只有大纲或要点，正文未写（乐、数当前就是这个状态）。
- 成稿：正文写完。判据三条——知识库对应文件齐备、按 §7 与 §11 校对通过、已回写《全书大纲一览.md》。
- 待渲染：成稿已登记，排队等渲染。
- 已渲染：走完 §9 的渲染流程和逐页检查。
- 重构废弃（旁支）：被重构方案取代的旧稿（如射卷 5→4 重构中退役的旧章），处理完即归档。
